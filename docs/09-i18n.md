# 09 — 多言語（日本語 / 英語）

## 方針

- 軽量に始める: **クライアント側の辞書 + Context** でラベルを切り替える（試作と同じ）。ロケールは Cookie に保存し、初回は `Accept-Language` から判定（`ja` 始まりなら ja、他は en）。ヘッダーのトグルで即切り替え。
- 作品名（products.title）は両言語で同じものを出す（作品名なので翻訳しない）。説明文が要る商品だけ `description_ja` / `description_en` を出し分ける。
- サイト名「the poster」（表記 THE POSTER）はロゴ画像で表示するため辞書には含めない。
- 将来 SEO 重視で URL を `/ja` `/en` に分けたくなったら `next-intl` のロケールルーティングへ移行できる。初版では不要。

## ロケール解決

```ts
// 優先順: Cookie 'locale' > Accept-Language > 'en'
// トグルで Cookie を書き換え、状態を更新して再レンダリング。
type Locale = 'ja' | 'en';
```

## Stripe Payment Element のロケール

Payment Element 自体も `locale` を渡せる（`elements({ locale })`）。サイトのロケールと合わせる（ja/en）。
