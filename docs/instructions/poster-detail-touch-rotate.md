# 詳細ページ：指（タッチ）で左右に自由回転できるようにする

## 背景・要望
竹内くんの要望：詳細ページで「指で左右の回転を操作したい」。
現状は Poster 内蔵の PresentationControls（snap=true）のため、指/マウスを離すと正面にスナップで戻ってしまい、自由に回し続けられない。

## 厳守事項
- 開発サーバーが PORT=3137 稼働中。**`npm run build` は絶対に実行しない**。検証は tsc / lint のみ。
- 「素のHTML風」方針（Tailwind不使用、例外CSSは app/globals.css）。
- drei v10。**一覧（トップ）の挙動・見た目は一切変えない**（ゆっくり自動回転・透かし・クリック遷移・SOLD OUT を維持）。
- 前回の「見切れ防止（回転しても枠内に収まる）」を維持すること。

## 要件（詳細ページの操作）
1. **指（タッチ）/マウスのドラッグで左右（水平）に自由回転できる**。左右はぐるぐる一周できる（水平360°制限なし）。
2. **指を離しても、その向きを保持する**（正面にスナップで戻らない）。
3. 上下方向は見やすい範囲に軽く制限（ポスターが裏返りすぎない程度。例：±20〜30度程度）。
4. ズーム「+ / −」ボタンは引き続き機能する。最も引いた（デフォルト）状態で、回転してもポスター全体が枠内に収まる（見切れ維持）。
5. 未操作時のふるまいは次のどちらかでよい（自然で実装が安定する方を選択）：
   - (推奨) 未操作時はゆっくり自動回転し、ユーザーが触ると自動回転は止まって手動操作になる。
   - もしくは自動回転なしの完全手動（指で回す）。
   どちらでも「指で左右に自由回転・離しても保持」が満たされていればよい。

## 実装方針（ヒント）
- 詳細ページ（`components/poster/PosterDetail.tsx`）では drei の **OrbitControls** を使うのが素直：
  - `enablePan={false}`、`enableDamping`（慣性で気持ちよく）、
  - 水平自由：`minAzimuthAngle`/`maxAzimuthAngle` は無制限（既定）、
  - 上下制限：`minPolarAngle`/`maxPolarAngle` を水平±20〜30度相当に設定、
  - ズーム：詳細の「+ / −」ボタンと整合させる。OrbitControls にズームを任せる（`minDistance`/`maxDistance` を前回の収まり範囲に）か、ボタンから OrbitControls の dolly を操作する。**見切れない最大引き距離（前回 D=11.0 相当）をデフォルトにする**。
  - 自動回転を残す場合は `autoRotate` + `autoRotateSpeed`（ゆっくり）。`onStart` でユーザー操作開始時に `autoRotate=false` にする。
- **二重制御を避ける**：Poster 内蔵の PresentationControls / idle 自動回転(mesh回転) と OrbitControls が競合しないようにする。
  - 案：`Poster` に `controls?: boolean`（または `interactive` モード）prop を足し、**詳細では Poster 内蔵コントロールと idle 回転を無効化**して、詳細側の OrbitControls だけで回す。一覧では従来通り（prop 既定値で挙動不変）。
  - Poster の透かし・テクスチャ・dispose・Float（控えめ化済み）等は維持。詳細で Float を残すと OrbitControls と干渉する場合は詳細のみ Float を切ってよい。
- タッチ操作のため Canvas の `touchAction: "none"`（設定済み）を維持。

## 受け入れ条件
1. 詳細ページで指/マウスのドラッグにより左右に自由回転でき、離しても向きが保持される（正面に戻らない）。
2. 上下は見やすい範囲に収まり、裏返りすぎない。
3. ズーム「+ / −」が機能し、デフォルト（引き）状態で回転しても見切れない。
4. 一覧（トップ）の挙動・見た目は従来通り（自動回転・透かし・クリック遷移・SOLD OUT）。
5. `npx tsc --noEmit` エラー0、`npm run lint` エラー0、consoleエラーを増やさない。build は実行しない。
