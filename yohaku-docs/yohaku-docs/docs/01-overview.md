# 01 — 概要 / スコープ

## 何を作るか

個人が自作した限定アートポスターを販売する、ミニマルなオンラインストア「Yohaku」。

## 必須要件

- **ストア（公開ページ）**
  - 真っ白な背景。左上にロゴ（ひし形マーク + "Yohaku"）。装飾は最小限。
  - 各ポスターが紙のように 360° 回る 3D 表示。ドラッグで回せる。
  - グリッド：PC は横 4 列、スマホは横 2 列。
  - 各商品に「残り N / 上限 M」を表示。在庫が減るとバーも減る。
  - 在庫が 0 になった商品はアーカイブ表示（販売終了・購入不可）。レコードは残す。
  - 日本語 / 英語の切り替え（ヘッダーのトグル）。
- **チェックアウト（購入者入力）**
  - Stripe Payment Element を使用。
  - 入力：メール、お届け先（氏名・住所・部屋番号(任意)・市区町村・都道府県・郵便番号）。**国フィールドは持たない**（Stripe 側が必要に応じて請求先国を聞く）。
  - 数量は在庫の残数を上限に選べる。
  - 注文サマリー（サムネイル・タイトル・数量・小計・送料・合計）。
- **管理画面（要認証・自分だけ）**
  - 商品の追加・編集・削除。
  - 商品はタイトル（作品名）と画像、価格、販売上限（= 限定枚数）を設定。
  - 公開 / 非公開（draft）トグル、手動アーカイブ。
  - 売上 / 注文の確認。
- **在庫の整合性**
  - 同時購入で上限を超えて売れない（オーバーセルしない）。
  - 在庫減算は決済完了（webhook）を受けてサーバー側でのみ実施。

## スコープ外（今回は作らない）

- クーポン / 割引コード、ユーザー登録・マイページ、レビュー、複数通貨、配送業者連携、メール配信基盤（Stripe の領収書メールに任せる）。
- これらは後から足せる設計にしておくが、初版には含めない。

## 用語

- **edition_size** … その商品の限定枚数（上限。例: 20）。
- **sold_count** … 売れた枚数。**残り = edition_size − sold_count**（残数はカラムを持たず計算で出す）。
- **status** … `draft`（非公開）/ `active`（販売中）/ `archived`（売り切れ or 手動で販売停止）。
- **locale** … `ja` / `en`。
- 金額は **cents（最小単位の integer）** で保持。表示時に通貨整形する。

## 注記（国フィールドについて）

フォームから国を外しても、Stripe の Payment Element はカードに応じて請求先の国を内部で収集するため決済は成立する。配送先として国を保存したくなったら、`orders.shipping_country` は用意済みなので後から入力欄を足すだけでよい。
